# 전투 시스템 구현 메모

## 기본 방향

- 전투 계산은 `BattleScene`이 아니라 `src/lib/battleState.ts`가 담당한다.
- `BattleScene`은 전투 상태와 이벤트를 화면에 보여주는 뷰 역할만 한다.
- 탭을 이동해 `BattleScene`이 사라져도 전투 루프는 전역 상태에서 계속 유지된다.
- 화면에 없을 때 발생한 과거 공격 애니메이션은 몰아서 재생하지 않는다. 전투 화면에 다시 들어온 뒤 발생하는 이벤트부터 연출한다.
- 전투 화면 재진입 시 `BattleScene`은 애니메이션 큐/플래그를 초기화하고 최신 전투 상태 스냅샷을 반영하되, 이미 진행 중인 전투 루프의 다음 턴 타이머를 리셋하지 않는다.
- 전투 시작 또는 새 웨이브 입장 연출 완료 직후에는 기다리지 않고 맨앞콩(`ally-1`)이 바로 첫 공격을 한다.
- 웨이브 전환이 필요한 상태에서는 전역 전투 phase를 `waiting-wave-transition`으로 두고, 전환 연출 완료 전까지 다음 공격 계산을 진행하지 않는다.
- 웨이브 전환이 시작되면 시작 시각을 기록한다. 전환 중 탭을 벗어났다가 돌아오면 경과 시간을 계산해 전진/페이드/입장 중간 상태부터 이어서 재생한다.

## 현재 수치

- 아군 콩 HP: `100`
- 아군 콩 공격력: `10`
- 적군 HP: `40`
- 적군 공격력: `0`
- 기본 턴 간격: `780ms`
- 웨이브 리셋 계산 지연: `220ms`
- 웨이브별 적 등장 수: `1~2마리` 랜덤

## 턴 순서

화면 배치 순서:

```txt
아군4 아군3 아군2 아군1 적군1 적군2 적군3 적군4
```

팀 턴 진행 순서:

```txt
아군 → 적군 → 아군 → 적군 → 반복
```

각 팀 안에서는 슬롯 순서대로 살아있는 유닛만 공격자로 순환한다.

```txt
아군: 아군1 → 아군2 → 아군3 → 아군4 → 반복
적군: 현재 웨이브에 등장한 적군만 슬롯 순서대로 반복
```

죽은 유닛은 같은 팀의 공격자 순환에서 스킵한다. 예를 들어 `적군1`, `적군2`가 등장한 웨이브에서 `적군1`이 죽으면 적군 턴의 공격자는 `적군2`가 된다.

웨이브가 바뀌면 팀 턴과 팀 내부 공격자 순서를 초기화한다. 새 웨이브의 첫 공격은 항상 맨앞콩(`ally-1`)이 한다.

## 타겟팅

- 아군은 가장 앞의 살아있는 적을 공격한다.
- 적군은 가장 앞의 살아있는 아군을 공격한다.
- 현재 적군 공격력은 `0`이므로 아군은 사실상 죽지 않는다.

## 웨이브 배치

- 각 웨이브의 적은 `1~2마리`가 랜덤하게 등장한다.
- 등장한 적은 기존 적 배치 영역 중앙을 기준으로 가운데 정렬한다.
- 적 사이 간격은 등장 수와 상관없이 일정하게 유지한다.
- 등장하지 않은 적 슬롯은 화면에 표시하지 않는다.

## 이벤트와 연출

`battleState.ts`는 실제 HP 감소와 사망 여부를 계산한 뒤 이벤트를 발행한다.

- `BattleAttackEvent`
    - 공격자
    - 대상
    - 데미지
    - 대상 현재 HP
    - 대상 최대 HP
    - 대상 사망 여부
- `BattleWaveResetEvent`
    - 적 전멸 후 새 웨이브 리젠 상태

`BattleScene`은 이벤트를 받아 다음을 연출한다.

- 근거리 공격 이동
- 공격자는 접근 완료 후 짧은 선딜을 두고 타격한 뒤, 짧은 후딜 후 원래 위치로 복귀한다.
- 아군 공격음 `dang.mp3`
- 적군 공격음 `kung.mp3`
- 데미지 텍스트
- 피격 시 아군 콩은 빨간색, 적군은 흰색 fill tint로 짧게 깜빡이고, 아군은 왼쪽/적군은 오른쪽으로 살짝 흔들림
- 캐릭터 발밑 HP바 갱신
- 적 사망 시 화면 밖으로 튕겨나간 뒤 숨김
- 적 전멸 시 아군이 오른쪽 화면 밖으로 퇴장 → 전투영역 검은색 페이드아웃 → 리젠 상태 반영 → 페이드인 → 아군이 왼쪽 화면 밖에서 원래 배치로 입장
- 웨이브 리셋 이벤트가 공격 애니메이션 중 도착하면 즉시 전환하지 않고, 현재 공격/사망 연출이 끝난 뒤 전환한다.
- 웨이브 전환 연출 중에는 전투 계산 루프를 일시정지하고, 입장 연출이 끝난 직후 맨앞콩이 딜레이 없이 공격하도록 즉시 재개한다.
- 전투탭 밖에서 웨이브 전환 상태가 되더라도 pending 웨이브 이벤트를 보관한다. 전투탭 복귀 시 전진 → 페이드아웃 → 페이드인 → 입장 연출을 먼저 처리하고, 완료 후 pending 이벤트를 consume한 다음 전투 루프를 재개한다.
- 웨이브 전환 도중 전투탭을 벗어난 뒤 복귀하면 전환 연출을 처음부터 반복하지 않고, 저장된 시작 시각 기준 경과 시간에 맞춰 중간 지점부터 이어서 재생한다.
- 웨이브 전환 중 발생한 공격 이벤트는 몰아서 재생하지 않고, 전환 완료 시 최신 전투 상태 스냅샷을 반영한다.
- 전투탭을 빠르게 벗어났다가 돌아와도 기존 다음 턴 타이머를 유지해 탭 전환으로 턴 간격이 앞당겨지지 않게 한다.

## 향후 확장 포인트

- 공격력/HP를 콩 유전자나 장비에서 계산하도록 변경
- 방어력, 치명타, 속성, 스킬 추가
- 적 웨이브별 스탯 증가
- 오프라인 진행 보상 계산
- 전투 결과 저장 및 재접속 복원
